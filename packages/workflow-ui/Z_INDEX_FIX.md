# WorkflowCanvas Z-index 层级修复

## 问题分析
WorkflowCanvas 中的连线（edges）被节点卡片（nodes）遮挡，影响了用户视觉体验和交互性。

## 解决方案
通过设置合理的 z-index 层级，确保：
1. 连线显示在节点下方
2. 节点端口（handles）具有最高优先级，确保连线能正确连接
3. 交互状态（选中、悬停、拖拽）具有适当的层级提升

## Z-index 层级结构
```
连线 (edges): z-index: 1
节点 (nodes): z-index: 2
端口 (handles): z-index: 3
选中/悬停连线: z-index: 4
拖拽节点: z-index: 5
连接线动画: z-index: 6
```

## 修改文件
- `/packages/workflow-ui/src/styles/globals.css`

## 设计原则
- **存在即合理**: 每个 z-index 规则都有其独特的用途
- **优雅即简约**: 简洁的层级结构，避免复杂的嵌套
- **性能即艺术**: 最小化重绘和重排
- **错误处理**: 确保所有交互状态都能正确显示

## 兼容性
- 与 React Flow 默认样式兼容
- 保持现有的交互效果（悬停、选中、拖拽）
- 不影响其他组件的样式

## 测试建议
1. 创建多个节点并连接
2. 验证连线显示在节点下方
3. 测试节点拖拽时的层级变化
4. 验证连线选中和悬停效果
5. 确保端口始终可点击和连接
