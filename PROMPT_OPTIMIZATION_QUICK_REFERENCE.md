# 提示词优化 - 快速参考卡片

## 一句话速记

**优雅的提示词 = 最少的字 + 最清的意 + 最强的效果**

---

## 四层结构（必背）

```
第 1 层：角色  ← 必须有
第 2 层：能力  ← 列出 3-5 个
第 3 层：原则  ← 列出 3-4 个（可选但强烈建议）
第 4 层：边界  ← 明确禁区（可选但很重要）
```

---

## 9 大技巧（快速应用）

| # | 技巧 | 怎么做 | 效果 |
|---|------|-------|------|
| 1 | 精简冗余词 | "在实际上" → "实际上" | -10% 字数 |
| 2 | 段落改列表 | 用 `-` 或 `•` 代替段落 | +60% 可读性 |
| 3 | 示例代替解释 | 一个例子胜千言 | -20% 字数，+40% 理解度 |
| 4 | 术语统一 | 全文只用一个名词 | -歧义 |
| 5 | 提取隐含意图 | 问"为什么"和"真正目标" | 瞄准本质 |
| 6 | 分层信息 | 核心→支撑→细节 | +扫一眼就懂 |
| 7 | 用格式代替词 | 列表、编号、标题 | +效率 |
| 8 | 编号原则非规则 | 原则生成规则，不列举规则 | 可维护性 ↑ |
| 9 | 单一真相源 | 定义一次，引用多次 | 易维护 |

## 常见反模式（避坑）

```
❌ "能做任何事"          → ✓ 列出具体 3-5 件
❌ 50+ 条规则            → ✓ 提炼为 3-4 个原则
❌ 平铺结构              → ✓ 分层：核心→支撑→细节
❌ 隐含假设              → ✓ 完整说明背景
❌ 过度优化（为短而短）   → ✓ "简洁 + 清晰"平衡
```

---

## 5 步优化流程（完整版）

```
1️⃣ 诊断（5 分钟）  ← 找症状
   └─ 模糊 / 冗余 / 结构混乱 / 意图不清

2️⃣ 分析（10 分钟） ← 找根因
   └─ 为什么模糊？重复能合并？结构能改？用户真想要什么？

3️⃣ 优化（20 分钟） ← 改改改
   └─ 精简→列表→标题→例子→分层

4️⃣ 验证（10 分钟） ← 问 5 个问题
   └─ 清晰？能力明确？有边界？没遗漏？像人话？

5️⃣ 测试（持续）   ← A/B 对比
   └─ 建立指标→对比新旧→持续迭代
```

## 发布前检查（15 项）

### 必检 ✓

```
清晰度：
  ✓ 第一句说清角色吗？
  ✓ 能 30 秒找到核心能力？
  ✓ 术语一致吗？

结构：
  ✓ 分层清晰吗？
  ✓ 导航线索够吗？

完整性：
  ✓ 角色、能力、原则、边界都有？
  ✓ 重要信息没遗漏？

质量：
  ✓ 朗读像人话吗？
  ✓ 没有冗余？
  ✓ 长度合理？
```

**通过所有检查 = 可以发布**

## 最小化模板（复制即用）

```markdown
## 角色
你是 [身份]，[核心特征]。

## 核心能力
- 能力 1：[说明]
- 能力 2：[说明]

## 工作原则
- 原则 1：[描述 + 为什么]
- 原则 2：[描述 + 为什么]

## 边界与禁区
- 不做：[具体禁区]
- 仅基于：[限制说明]
```

## 3 个黄金问题

### 问题 1：怎么知道什么时候停止优化？
**答：** 当你能用 30 秒讲清核心时，停止。剩下的是消遣。

### 问题 2：怎么平衡简洁和完整？
**答：** 删掉一个词，如果内容缺少了 = 这词很重要；如果更清晰了 = 这词是噪音。

### 问题 3：怎么判断优化是否成功？
**答：** 用户搞不清时，会问"这是什么意思"；优化好的提示词，用户很少有这个困惑。

## 5 个常见陷阱

| 陷阱 | 表现 | 如何避免 |
|------|------|--------|
| **模糊角色** | "有帮助的助手" | 具体身份 + 核心特征 |
| **冗长列表** | 50+ 条规则 | 提炼为 3-4 个原则 |
| **假谦逊** | "可能不完美但尽力" | 明确说优势和限制 |
| **隐含假设** | "你应该知道..." | 完整说明背景 |
| **无序堆积** | 没有章节，没有结构 | 用标题和列表分层 |

## 提示词优化的三个阶段

### 初级（入门者）
- 学会删除冗余词
- 段落改成列表
- 添加章节标题

### 中级（进阶者）
- 提取隐含意图
- 分层信息密度
- 用原则代替规则

### 高级（大师级）
- 审视整体架构
- 重新设计结构
- 让形式适应内容

## 持续优化的循环

```
优化完成
  ↓
监控 1-2 周
  ↓
  有显著问题吗？
  ├─ 是 → 调整 → 重新测试 → 监控
  └─ 否 → 标记为稳定 → 应用到其他角色
```
